Этапность внедрения

Никаких кусков деплоя: сначала подтверждаем модель, потом внедряем цельно и тестируем цепочки.

0. Без привязки

Документ, демо-экран и контракт данных. Основной код, API и база не трогаются.

1. Контракт

`contract.v1.json` описывает, что будущий агрегатор должен вернуть в карту.

2. Изолированный UI

Отдельный модуль карты на моках. В `App.js` не подключаем до решения.

3. Read-only API

`/projects/director-map` появится позже и сначала будет только читать данные.

4. Проверка связей

Все неуверенные совпадения идут в `needs_review`, без автопроведения.

5. Действия

Открыть ЖПР, АОСР, заявку снабжения, оплату или задачу. Карта не запрещает и не проводит сама.

6. Smoke-цепочки

Смета, ЖПР, склад, накладные, документы, роли и финансы проходят проверку.

7. Внедрение разом

После проверки включаем через feature flag как цельную функцию.

Главное правило: пока карта собрана без привязки. После внедрения она подсказывает и связывает, но не ставит жесткий стоп. Она не должна сама проводить склад, создавать фактические деньги или молча связывать спорные документы.
