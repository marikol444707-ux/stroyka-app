# Инструкция: Директор

## Назначение роли

Директор видит всю компанию: объекты, сметы, материалы, снабжение, финансы, персонал, документы, аналитику и ИИ-контроль. Это роль для принятия решений, а не для ежедневного ручного ввода всех фактов.

## Первый вход

1. Сменить временный пароль.
2. Проверить кнопку `Статус`: backend, база и S3 должны быть `ok`.
3. Создать первый реальный объект.
4. Создать пользователей: прораб, сметчик, бухгалтер, снабженец, кладовщик, мастера.
5. Назначить проектные роли на объект.

## Ежедневная работа

- Смотреть главный дашборд.
- Открывать `ИИ-контроль` и разбирать риски.
- Проверять задолженности, поставки, материалы и работы вне сметы.
- Утверждать управленческие решения по допработам, закупкам и оплатам.
- Смотреть сопоставительные ведомости смет перед изменением договора.

## Что должен видеть результатом

- Новый объект появляется в списке объектов.
- Назначенные роли видят только свои объекты.
- Задачи ИИ-контроля имеют ответственного, причину и переход в нужное место.
- Финансовый прогресс сходится с оплатами, актами и счетами.

## Нельзя делать

- Не менять старую договорную смету задним числом.
- Не заносить одну работу одновременно вручную в смету и отдельно в ЖПР.
- Не списывать материалы вместо мастера без понятной причины.
- Не держать временный пароль после запуска.

## Быстрая проверка

Открыть `Объекты`, `Сметы`, `Снабжение`, `Склад`, `Бухгалтерия`, `Пользователи`, `ИИ-контроль`, `Статус`. Все разделы должны открываться без ошибок.
