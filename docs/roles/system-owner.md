# Инструкция: Владелец платформы, system_owner

## Назначение роли

`system_owner` обслуживает SaaS-платформу: компании, тарифы, демо-заявки, платежи платформы и системный статус. Это не рабочая роль строительного объекта.

## Первый вход

1. Открыть кабинет владельца платформы.
2. Проверить список компаний.
3. Проверить тарифы и демо-заявки.
4. Проверить `Статус системы`.

## Ежедневная работа

- Создавать компанию-клиента.
- Выдавать инвайт директору компании.
- Менять тариф, срок доступа и статус компании.
- Смотреть системные ошибки.
- Помогать поддержке, если у клиента проблема с доступом.

## Что должен видеть результатом

- Компания создана.
- Инвайт выдан директору компании.
- Тариф и срок действия видны в кабинете.
- Системные ошибки можно посмотреть без доступа к секретам.

## Нельзя делать

- Не вести ЖПР клиента.
- Не списывать материалы.
- Не менять сметы и финансы клиента без задачи поддержки.
- Не хранить пароли клиентов в открытом виде.

## Быстрая проверка

Открыть `system_view`: обычный рабочий объект не должен быть основной точкой входа.
