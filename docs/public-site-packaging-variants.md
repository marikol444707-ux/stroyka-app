# Варианты упаковки публичного сайта

Документ нужен для сравнения подач сайта перед визуальной доработкой через Creative Production или ручной дизайн.

## Вариант 1. Директорский контроль

Смысл: показать, что заказчик получает не просто бригаду, а управляемый объект.

- Первый экран: строительство с понятной сметой и контролем директора.
- Главный акцент: смета, сроки, отклонения, журнал, уведомления.
- Доказательства: статус объекта, фото работ, акты, материалы, гарантия.
- Для кого: заказчик, который боится потери контроля и скрытых расходов.

## Вариант 2. Живой объект

Смысл: показать объект как прозрачную ленту событий.

- Первый экран: объект в работе, прогресс, этап, фото за день.
- Главный акцент: что сделано сегодня, что закуплено, кто отвечает.
- Доказательства: фотоотчёты, скрытые работы, поставки, выдача материалов.
- Для кого: заказчик, которому важно видеть стройку без ежедневных звонков.

## Вариант 3. Документы и бухгалтерия

Смысл: показать сильную сторону в первичке и закрывающих документах.

- Первый экран: стройка без потерянных счетов, накладных и актов.
- Главный акцент: `счет -> накладная -> склад -> акт -> оплата`.
- Доказательства: сканы накладных, привязка к поставщикам, проверка оплат.
- Для кого: коммерческие заказчики, руководители и бухгалтерия клиента.

## Вариант 4. Полная цепочка ERP

Смысл: показать, что компания ведёт весь цикл в одной системе.

- Первый экран: от сметы до гарантии в одном цифровом паспорте.
- Главный акцент: `смета -> заявка -> счет -> накладная -> склад -> выдача -> работы -> акты -> оплата`.
- Доказательства: калькулятор, галерея работ, паспорт объекта, CRM-заявка.
- Для кого: заказчик, который выбирает не самую дешёвую бригаду, а управляемого подрядчика.

## Рекомендация

Для текущего сайта базовым оставить вариант 4, а на первом экране усилить вариант 1:
понятная смета, контроль директора и цифровой паспорт объекта. Вариант 2 использовать в блоке
`Наши работы`, вариант 3 — в блоке для коммерческих клиентов и бухгалтерии.
