# backend/features/project_launch

Отдельный backend-модуль запуска объекта.

Задача модуля: принять документы объекта, распознать их, сохранить AI-черновик, показать пользователю недостающие данные и только после подтверждения записать данные в рабочие таблицы.

## Границы модуля

Модуль отвечает за:

- `project_launch_drafts`
- `project_contract_terms`
- подготовку `counterparties`
- запуск распознавания через существующий `document_recognition`
- создание `ai_findings` и `ai_tasks` по договору
- подготовку черновика сметы для `estimates`

Модуль не отвечает за:

- складские приходы и списания;
- оплату и бухгалтерское проведение;
- автоматическую активацию сметы;
- закрытие, архивирование или удаление объекта;
- SaaS-кабинет владельца платформы.

## Файлы

- `routes.py` - безопасный первый слой FastAPI endpoints `/project-launch/*`: черновики и готовность объекта.
- `schema.py` - SQL init для таблиц модуля.
- `service.py` - общие функции нормализации, проверки доступа к объекту и сериализации черновиков.
- `extractors.py` - нормализация результата `document_recognition`.
- `legal_review.py` - правила юридического контроля договора.

Модуль подключается в `backend/main.py` только тонкой регистрацией. Применение черновика в рабочие сущности не включено до отдельного ручного подтверждения.
