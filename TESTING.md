# TESTING — план проверки stroyka-app

Полное пошаговое руководство для проверки системы и поиска багов.

Время на полное прохождение: **4-6 часов**. Можно разбивать на сессии по 30-60 минут.

---

## 0. Подготовка

### 0.1 Тестовые аккаунты

Для проверки разных ролей войдите под этими логинами по очереди:

1. `admin@stroyka.ru` / `admin123` — **директор** (полный доступ)
2. `prorab@stroyka.ru` / `prorab123` — **прораб**
3. `master@stroyka.ru` / `master123` — **мастер**
4. `buh@stroyka.ru` / `buh123` — **бухгалтер**
5. `zxc@mail.ru` / `12345` — мастер (создан вручную)

Если нужно создать аккаунт для технадзора/заказчика/субподрядчика — войдите под директором → Персонал → создать сотрудника с системной ролью.

### 0.2 Что иметь под рукой

1. Telegram/блокнот, куда писать найденные баги
2. Возможность делать скриншоты (Cmd+Shift+4 на маке)
3. Открытую консоль браузера (F12 → вкладка Console) — там видны ошибки JavaScript
4. Excel-файл с тестовой сметой LSR/Grand Smeta (если хотите проверить импорт)
5. Любая фотография (jpg/png) — для тестов прикрепления

### 0.2.1 Временный замдиректора для прогона роли

Если нужно быстро проверить права роли `зам_директора` без ручного создания через интерфейс, на сервере есть готовая схема:

1. Создать или обновить временного пользователя:

```bash
cd /var/www/stroyka-app && python3 scripts/manage-temp-user.py create --email qa.zamdir@stroyka26.pro --name "QA Зам директора" --password 'TempZam!2026' --role зам_директора --project 'Кисловодск Лицей 4'
```

2. Прогнать доступ этой роли по основным API:

```bash
cd /var/www/stroyka-app && SMOKE_EMAIL='qa.zamdir@stroyka26.pro' SMOKE_PASSWORD='TempZam!2026' bash scripts/role-smoke-check.sh
```

3. После проверки отключить временный доступ:

```bash
cd /var/www/stroyka-app && python3 scripts/manage-temp-user.py disable --email qa.zamdir@stroyka26.pro
```

Скрипт не удаляет историю пользователя, а только отключает вход. Это безопаснее для аудита.

### 0.3 Как фиксировать баги

Когда что-то работает не так, запишите 4 вещи:

1. **Где** — путь в системе (например: «Бухгалтерия → По объектам → выбрал Лермонтова → нажал КС-2»)
2. **Что нажимал** — последовательность действий
3. **Что ожидал** — как должно быть
4. **Что получилось** — что вышло вместо
5. **Скриншот** — обязательно

Также проверьте F12 → Console — есть ли красные ошибки. Скопируйте текст красной ошибки.

## 1. Базовая проверка (все роли) — 10 мин

### 1.1 Вход в систему

1. Откройте https://stroyka26.pro
2. Введите `admin@stroyka.ru` / `admin123`
3. Нажмите «Войти»

✅ **Ожидание:** перешли на главную страницу (Dashboard). Слева тёмный сайдбар.
⚠️ **Если:** ошибка входа / зависание / пустая страница — баг.

### 1.2 Выход

1. Наведите курсор на левый край экрана → выедет сайдбар
2. Внизу сайдбара кнопка «Выйти» (красная)
3. Нажмите

✅ **Ожидание:** вернулись на экран входа.

### 1.3 Сайдбар (левое меню)

1. Войдите снова под admin
2. Наведите курсор на левый край

✅ **Ожидание:** сайдбар выезжает, цвет ТЁМНО-СИНИЙ (не прозрачный!), текст белый и читается. Видна иконка СтройКа сверху, ваше имя ниже, список разделов.
⚠️ **Если:** сайдбар прозрачный или белый — обязательно сделайте Cmd+Shift+R (жёсткое обновление), потом проверьте снова.

### 1.4 Переключатель темы

1. В правом верхнем углу шапки — иконка ☀️/🌙
2. Нажмите её

✅ **Ожидание:** вся страница меняет цвет на тёмную (или светлую если была тёмной). Иконка тоже меняется.
⚠️ **Проверьте:** все тексты должны оставаться читаемыми. Если что-то стало невидимым — баг + скриншот.

3. Обновите страницу (F5)

✅ **Ожидание:** тема сохранилась (запомнилась в браузере).

### 1.5 Уведомления

1. В правом верхнем углу — иконка колокольчика (Bell)
2. Если есть цифра — нажмите

✅ **Ожидание:** появилось окно со списком уведомлений или «Нет уведомлений». Кнопка «Прочитать все».

### 1.6 Глобальный поиск

1. В шапке поле «Поиск»
2. Введите название любого проекта или сотрудника (от 2 букв)

✅ **Ожидание:** появляется выпадающий список с результатами. Клик переводит на соответствующую страницу.

#### 2.1.2 Редактирование

2. Измените бюджет
3. Сохраните

✅ **Ожидание:** новый бюджет отображается. **Это важный тест** — я недавно чинил баг в этом месте (PUT /projects).

#### 2.1.3 Архив и восстановление

1. На карточке проекта — стрелка вниз → «Архивировать»
2. Кнопка «Архив» вверху списка
3. Найдите архивный проект → «Восстановить»

✅ **Ожидание:** проект вернулся в активные.

#### 2.1.4 Удаление

1. Создайте тестовый проект «УДАЛИТЬ_ТЕСТ»
2. Иконка корзины → подтвердите

✅ **Ожидание:** проект исчез из списка.

### 2.2 Клиенты — 5 мин

1. Сайдбар → «Клиенты»
2. «+ Новый клиент» — заполните ФИО, телефон, email
3. Сохраните
4. Отредактируйте
5. Удалите

✅ **Ожидание:** все три операции работают.

### 2.3 Персонал — 20 мин

#### 2.3.1 Сотрудники

1. Сайдбар → «Персонал»
2. Вкладка «👥 Сотрудники»
3. «+ Добавить сотрудника» → заполните развёрнутую форму:

   - **Доступ в систему:** включите, выберите роль (например прораб), email и пароль (или все три, или ничего)
   - **Документы:** паспорт, ИНН, СНИЛС

4. Сохраните

✅ **Ожидание:** сотрудник появился в списке. Если включали «доступ» — также появился в списке пользователей.
⚠️ **Тест валидации:** попробуйте сохранить с одним только email без пароля — должна быть ошибка.

#### 2.3.2 Раскрываемая карточка

1. Кликните на строку сотрудника
2. Должна раскрыться сводка: 4 карточки (занятость, паспорт, ИНН, банк) + договоры, акты, согласия ПД, инструктажи ТБ

✅ **Ожидание:** всё видно. Если нет данных — заглушка «Документов нет».

#### 2.3.3 Прочие документы

1. В раскрытой карточке → «Прочие документы» → «Добавить»
2. Тип, файл, дата
3. Сохраните

✅ **Ожидание:** документ появился.

#### 2.3.4 Табель

1. Вкладка «📅 Табель»
2. Выберите месяц

#### 2.3.5 Сдельные

1. Вкладка «💵 Сдельные»

✅ **Ожидание:** список выплат за сделанную работу.

1. Откройте раздел «Сметы»
2. Найдите вкладку «Прайс-листы»

✅ **Ожидание:** появился прайс с 5-15 позициями. **Внимание на читаемость кнопки во время ожидания** — текст белый на зелёном.
