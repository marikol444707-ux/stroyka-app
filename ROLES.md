# ROLES.md — карта ролей и доступов stroyka-app

Документ для анализа: кто что видит и что может делать в системе.

---

## 1. Директор / Зам директора

### Видит в боковом меню
- Главная (дашборд)
- Проекты / Объекты
- Сметы и прайс-листы
- Склад
- Персонал
- Бухгалтерия (10 вкладок)
- Поставщики
- Чат
- Аналитика
- CRM
- Настройки

### Может вносить и редактировать
- Создаёт / удаляет / архивирует проекты
- Все сметы и прайс-листы
- Распределяет позиции сметы по бригадам
- Принимает на работу / увольняет сотрудников
- Выдаёт системные доступы (роли)
- Все печатные формы (20 шт)
- Гарантийные условия проекта
- Замечания ГСН по любому объекту
- Создаёт коды приглашений

### Утверждает и подписывает
- АОСР (как генподрядчик — одна из 4 подписей)
- Непредвиденные работы (доп.соглашения)
- Наряды бригадам (статус «Подписан»)
- Авансовые отчёты сотрудников
- Возмещение «Моих трат»
- Счета поставщиков

### Видит данные
- Бюджеты всех проектов
- Зарплаты всех сотрудников
- Аудит-лог системы
- Полный склад и движения

## 2. Бухгалтер

### Видит в меню
- Главная
- Бухгалтерия (полная)
- Персонал
- Чат
- Настройки

### НЕ видит
- Проекты для редактирования (только через Бухгалтерию)
- Сметы и прайс-листы
- Склад

### Может вносить
- Платежи (project_payments)
- Записи зарплат, акты
- Подотчётные платежи
- Регистрация счетов поставщиков
- Утверждение авансовых отчётов

### Утверждает
- Авансовые отчёты
- Счета поставщиков → оплата
- АОСР к оплате (после подписи 4 сторон)

### Видит данные
- Все финансовые движения
- Зарплаты сотрудников
- Аудит-лог
- 20 печатных форм

---

## 3. Прораб

### Видит в меню
- Главная
- Проекты (свои)
- Склад (выдача материалов, приёмка)
- Поставщики
- Персонал (своей бригады)
- Журналы (ЖПР, ТБ)
- Документы проекта
- Чат
- Погода

### Может вносить
- Журнал производства работ (work_journal)
- Приёмка накладных (вручную или сканом)
- Передача материалов мастерам
- Замечания ГСН
- Создание АОСР (через смету: ставит 🔒 на позицию)
- Непредвиденные работы (на согласование)
- Инструктажи ТБ
- Авансовые отчёты
- «Мои траты» с возмещением
- Подпись АОСР как «Подрядчик»
- Редактирование условий гарантии своего объекта

### НЕ может
- Создавать проекты
- Видеть зарплаты других
- Бухгалтерию открыть
- Утверждать непредвиденные

---

## 4. Мастер / Субподрядчик

### Видит в меню (узкий)
- Работы (главная страница — автоопрытие)
- Материалы (выданные ему)
- Документы (свои: договор, акты, согласие ПД)
- Чат

### Может вносить
- «Сделано» в позициях сметы (мои работы по смете)
- Свободные работы через прайс (если нет наряда)
- Материалы использованные в работе
- Непредвиденные работы (на согласование)
- Авансовые отчёты
- «Мои траты»

### НЕ может
- Видеть другие проекты, других мастеров
- Бухгалтерию, склад целиком, сметы целиком
- Утверждать никого

### Особенности
- При входе сразу попадает на страницу «Работы»
- Видит только позиции сметы где `brigade === user.name || user.brigade`
- Получает материалы от прораба, подписывает приёмку
- Подписывает АОСР как «Субподрядчик»

## 7. Технадзор

- Записи в кабельный журнал

### НЕ может
- Создавать / редактировать проекты
- Финансы, склад, сметы
- Утверждать непредвиденные

## 8. Заказчик

### НЕ видит

- Сметы (только своего объекта в Документах)
- Персонал

# ⚠️ Текущие проблемы доступа

## 1. «Мои траты» доступны только директорам
Кнопка «⚡ Быстро» только на дашборде. Мастера, прорабы, кладовщики НЕ могут внести свои траты — они вообще не видят кнопку.

**Кому должна быть доступна:** всем кто тратит свои деньги на стройке (прораб, мастер, снабженец, кладовщик).

## 2. «Расход по объекту» открыт всем
Любой кто открыл проект может внести ручной расход. Это рискованно.

**Кому должна быть доступна:** только финансовые роли (директор / зам / бухгалтер) + прораб для своего объекта.

## 3. «Подотчётные платежи» — выдать может любой
Сейчас любой может «выдать подотчёт» сам себе, что некорректно.

**Кому должна быть доступна:** только бухгалтер / директор.

## 4. «Передача мастеру» — открыта всем
Любой может списать материалы со склада.

**Кому должна быть доступна:** прораб, кладовщик.

## 5. «Создать наряд бригаде» — открыто всем
Любой может создать наряд (статус «Черновик»).

**Кому должна быть доступна:** только директор / зам / прораб.

## 6. «Непредвиденные работы» — добавляет любой
Это правильно: мастер фиксирует факт необходимости. Но утверждает только директор. Логика верна.

## 7. Внутри проекта роль фильтрация слабая
Прораб видит ВСЕ проекты, а должен только свой (где он foreman или project_name).

## 8. Сметы — мастер не должен видеть всё
Сейчас мастер видит сметы целиком если зашёл на страницу. Должен только свои позиции «по смете».

# 💡 Что я предлагаю обсудить

После того как вы изучите этот документ, нужно решить:

1. **Ограничить кнопку «⚡ Быстро» только директорами**, или РАСШИРИТЬ её на других? (вынести «Мои траты» в самостоятельный пункт меню?)

2. **«Расход по объекту»** — ограничить только финансами + прорабом своего объекта?

3. **Подотчёт** — ограничить выдачу только бухгалтером?

4. **Передача мастеру** — ограничить прорабом и кладовщиком?

5. **Создание нарядов** — ограничить директором и прорабом?

6. **Прораб видит только свой объект?** Сейчас видит все. Это нужно проверить.

7. **Аудит-лог открыт бухгалтеру** — нужно ли? Возможно только директору?

8. **«Мои траты» с категорией** — нужны ли отдельные права на категории? (например мастер не может категоризировать как «Аренда техники»)

После вашего решения — внесу правки в код.
